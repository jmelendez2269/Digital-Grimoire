# Password Reset Flow Documentation

## Overview

The Digital Grimoire implements a secure password reset flow using Supabase Auth. This document outlines the complete flow, configuration requirements, and user experience.

---

## Flow Architecture

### User Journey

```
1. User clicks "Forgot your password?" on login page
   ↓
2. User enters their email on /forgot-password
   ↓
3. Supabase sends a magic link to user's email
   ↓
4. User clicks the link in their email
   ↓
5. User is redirected to /reset-password with session token
   ↓
6. User enters and confirms new password
   ↓
7. Password is updated, user redirected to login
```

## Security Features

### 1. Token-Based Authentication
- Reset links contain secure tokens generated by Supabase
- Tokens expire after a configurable time period
- One-time use tokens (invalid after password update)

### 2. Email Privacy
- System doesn't reveal whether an email exists
- Generic success message shown for all email entries
- Prevents email enumeration attacks

### 3. Password Requirements
- Minimum 8 characters
- Client-side validation
- Server-side validation via Supabase
- Visual feedback on requirements

### 4. Session Validation
- Checks for valid recovery session before allowing password reset
- Shows error if link expired or invalid
- Provides option to request new reset link

## Supabase Configuration

### Required Setup

#### 1. Email Templates
Configure the password reset email template in Supabase Dashboard:

**Path:** `Authentication > Email Templates > Reset Password`

**Recommended Template:**
```html
<h2>Reset Your Password</h2>
<p>Someone requested a password reset for your Digital Grimoire account.</p>
<p>Click the link below to reset your password:</p>
<p><a href="{{ .ConfirmationURL }}">Reset Password</a></p>
<p>If you didn't request this, you can safely ignore this email.</p>
<p>This link expires in 1 hour.</p>
```

#### 2. URL Configuration
Configure redirect URLs in Supabase Dashboard:

**Path:** `Authentication > URL Configuration`

Add to **Redirect URLs:**
- `http://localhost:3000/reset-password` (development)
- `https://yourdomain.com/reset-password` (production)

#### 3. Email Rate Limiting
Recommended settings to prevent abuse:

**Path:** `Authentication > Rate Limits`

- **Password Reset Requests:** 3 per hour per email
- **Failed Attempts:** 5 per hour per IP

#### 4. Token Expiration
Configure token lifetime:

**Path:** `Authentication > Settings`

- **Magic Link Expiry:** 3600 seconds (1 hour) - recommended
- Adjust based on security requirements

## Error Handling

### Common Errors & Solutions

#### "Invalid or expired reset link"
**Cause:** Token expired or already used  
**Solution:** Request new reset link via `/forgot-password`

#### "Passwords do not match"
**Cause:** Password and confirmation don't match  
**Solution:** Re-enter passwords carefully

#### "Password must be at least 8 characters"
**Cause:** Password too short  
**Solution:** Use longer password

#### Network Errors
**Cause:** Connection issues  
**Solution:** Check internet connection and retry

## Troubleshooting

### Email Not Received

**Check:**
1. Spam/junk folder
2. Email address is correct
3. Supabase email service is configured
4. Rate limits not exceeded
5. Email provider settings in Supabase

**Solution:**
- Verify Supabase email settings
- Check Supabase dashboard logs
- Consider using custom SMTP provider

### Reset Link Not Working

**Check:**
1. Link hasn't expired
2. Link hasn't been used already
3. Redirect URL is configured in Supabase
4. Token is present in URL

**Solution:**
- Request new reset link
- Verify Supabase redirect URL configuration

### Password Update Fails

**Check:**
1. Valid session exists
2. Password meets requirements
3. Network connection is stable

**Solution:**
- Ensure token is valid
- Check browser console for errors
- Verify Supabase connection

## API Reference

### Supabase Auth Methods Used

#### `resetPasswordForEmail(email, options)`
Sends a password reset email.

**Parameters:**
- `email` (string): User's email address
- `options.redirectTo` (string): URL to redirect after clicking reset link

**Returns:** Promise with error if any

#### `getSession()`
Retrieves the current session.

**Returns:** Promise with session data

#### `updateUser(attributes)`
Updates user attributes.

**Parameters:**
- `attributes.password` (string): New password

**Returns:** Promise with updated user data or error
