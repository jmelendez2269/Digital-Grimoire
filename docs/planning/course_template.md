# Project Parallax — Course Production Template
## Version 1.0

> **Purpose:** This is the standardized template for every course in the Prismatic Learning curriculum. Every field must be completed before a course enters production. The template enforces the decisions, naming conventions, and structural principles established across all working sessions.

---

## COURSE-LEVEL FIELDS

### Course Metadata

| Field | Description | Example |
|-------|-------------|---------|
| `course_id` | Unique identifier (C##) | C01 |
| `title` | Course name | How Humans Know What They Know |
| `core_question` | The engine — a question, never a topic | "What counts as truth — and who decides?" |
| `arc` | Which curriculum arc this belongs to | Foundational Synthesis / Transformation & Inner Work / Convergence & Modern Application |
| `arc_position` | Position within the arc (1-5) | 1 |
| `length_weeks` | Number of weeks (6–8) | 8 |
| `level` | Difficulty / prerequisite level | Foundational–Intermediate |
| `orientation` | Positioning between rigor and insight | Bridge between academic rigor and esoteric insight |
| `mode` | Experiential tone | Reflective, analytical, initiatory |

### Course Premise
> 2–4 sentences. Why this course exists. What gap it fills. Written in the platform voice — calm, precise, invitational. Not a sales pitch.

### Key Tensions / Lenses in Dialogue (Course Spine)
> 3–6 named tensions that run through the entire course. Format: **A** vs **B**. These are the fault lines the course walks along. Each weekly question should connect to at least one of these.

### Learning Outcomes
> What the seeker can **do** or **see differently** by the end. 4–6 bullet points. Use active verbs (distinguish, identify, explain, recognize, build). Not "understand" — that's unmeasurable.

### Tone & Safety Note
> Brief statement on what this course does NOT do (e.g., does not induce altered states). Grounds expectations.

### Completion Pathways
> What courses this prepares the seeker for. Establishes the curriculum as a connected system.

## WEEKLY FIELDS (Weeks 1 through N-1)

Each non-capstone week contains the following fields, all required:

---

### Week [N] — [Title]

#### Core Question
> A single question. This is the engine of the week. It is never a topic.

#### Key Tension
> Format: **A** vs **B** — the specific fault line this week explores. Should connect to one or more of the course-level tensions.

#### Lens Focus
> Which lenses are primary this week (2–3 lenses). Name them explicitly. All 7 remain available, but these are foregrounded.

#### Readings (Selections)
> 2–4 texts selected from the platform library. The number is determined by the question — not fixed. The test for inclusion: "Does this text introduce a genuinely distinct position on this week's question?"

For each reading:

| Field | Description |
|-------|-------------|
| `text_title` | Name of the source work |
| `selection_rationale` | 1–2 sentences: why this text was chosen, what tension it introduces |
| **The Keystone** | The essential fragment. If the seeker reads nothing else, read this. A paragraph, maybe two. The seed of the inquiry. Specify: chapter/section/verse + brief description of the passage. |
| **The Passage** | The working reading. Enough context to engage meaningfully with the week's question and do the lens work. A few pages. Specify: chapter range or section boundaries. |
| **The Full Text** | The complete chapter or section. For seekers who want to sit with the source. Specify: full chapter/section reference. |

#### Lens Exercise
> A specific, actionable exercise that practices moving between perspectives. Not "reflect on..." — a concrete task with a defined output. This is the product's secret weapon — never make it optional or compressed.

| Field | Description |
|-------|-------------|
| `prompt` | The exercise instruction |
| `instructions` | Step-by-step guidance if needed (optional, for complex exercises) |

#### Synthesis Prompt
> Integration, not summary. A question that forces the seeker to hold multiple perspectives without collapsing them.

| Field | Description |
|-------|-------------|
| `prompt` | The synthesis question |
| `expansion` | 2–4 guiding sub-questions, sentence stems, or a suggested starting approach to help the seeker get started |

#### Convergence Micro-Artifact
> A small, defined output that captures this week's synthesis work. Each micro-artifact rolls up into the capstone.

| Field | Description |
|-------|-------------|
| `name` | What it's called (e.g., "Truth Inventory", "Model vs Meaning Chart") |
| `description` | What it is — format, scope, length |
| `purpose` | What it does — how it supports synthesis |
| `capstone_connection` | How it feeds into the final capstone artifact |

## CAPSTONE WEEK (Final Week)

The capstone week has a different structure:

### Week [N] — [Title] (Capstone)

#### Core Question
> The integrative question that pulls all prior weeks together.

#### Capstone Artifact
> The major synthesis output for the course.

| Field | Description |
|-------|-------------|
| `name` | What it's called |
| `description` | What it contains, its structure |
| `components` | Which micro-artifacts from prior weeks feed into it |
| `purpose` | What building it teaches the seeker |

#### Final Reflection
> A closing question or prompt. Not a summary — a door that opens forward.

#### Completion Pathways (Reiterated)
> What courses this prepares the seeker for. Same as course-level field, surfaced again at the moment of completion.

## TEMPLATE DESIGN PRINCIPLES

These are not fields — they are constraints that govern how fields are filled:

1. **Questions before topics.** Every Core Question is a genuine question, not a topic dressed as one. "Alchemy" is a topic. "Can transformation be systematized?" is a question.

2. **Texts create tension, not illustration.** Readings are selected because they disagree with each other, not because they all support the week's theme from different angles. The lenses handle multi-perspective work — the texts create friction.

3. **2–4 readings per week, driven by the question.** Minimum 2 (you need tension). Maximum 4 (cognitive ceiling). The test: "Does adding this text introduce a genuinely distinct position, or is it restating something another text already covers?"

4. **Three reading tiers per text.** Every selected text gets a Keystone (essential fragment), Passage (working reading), and Full Text (deep reference). Seekers self-select their depth.

5. **Lens Exercises are the product's secret weapon.** They are never optional, never compressed, never vague. A Lens Exercise has a defined output.

6. **Synthesis Prompts have teeth.** They create productive discomfort. "What breaks when...?" not "What did you learn about...?"

7. **Micro-Artifacts roll up.** Every weekly output feeds the capstone. Nothing is throwaway.

8. **Equanimity over conflict.** When lenses meet, the course holds convergences and divergences with equal attention. Neither pattern is privileged. Week titles use "When Lenses Meet" not "Conflict Between Lenses."

9. **No blind spots.** Each lens sees fully. The capstone maps "lenses you use less naturally," not "blind spots."

10. **Transformation is structural, not sensational.** No induced altered states. No destabilization. Grounding, reflection, discernment.
